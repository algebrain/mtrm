# `mtrm` application

`mtrm` — личный терминальный менеджер для локальной работы.

Этот документ описывает именно исполняемый пакет `app`: его жизненный цикл, горячие клавиши и служебные файлы.

## CLI поведение

Исполняемый файл поддерживает три прямых режима до запуска TUI:

- `mtrm --help`
- `mtrm --version`
- `mtrm --debug-log PATH`

`--help` и `--version` печатают информацию и завершаются без входа в raw mode и без запуска alternate screen.

`--debug-log PATH` запускает обычный `mtrm`, но дополнительно пишет сырые чанки PTY в указанный файл. Это предназначено для диагностики проблем терминальной эмуляции и интерактивных fullscreen приложений.

## Жизненный цикл приложения

1. При запуске `main()` определяет оболочку и текущий каталог.
   Если переданы `--help` или `--version`, приложение завершается раньше и TUI не поднимается.
   Оболочка по умолчанию запускается в интерактивном режиме.
2. Затем `App::restore_or_new()` пытается загрузить снимок состояния через `mtrm-state`.
3. Если файла состояния нет, создается новое приложение с одной вкладкой и одним окном.
4. Если файл состояния есть, `mtrm-tabs` восстанавливает вкладки и раскладки, запуская новые процессы оболочки в сохраненных каталогах.
5. При старте также загружается `~/.mtrm/keymap.toml`, а если его нет, он создается автоматически из встроенного шаблона.
6. После запуска `run()` входит в главный цикл:
   - читает клавиатурные события;
   - преобразует их через `mtrm-input`;
   - отправляет команды в `mtrm-tabs` или байты в активный PTY;
   - сохраняет состояние;
   - строит `FrameView` и отрисовывает кадр через `mtrm-ui`.

## Горячие клавиши

- `Ctrl+C` — копировать выделенный текст активного окна в системный буфер обмена.
- `Ctrl+V` — вставить текст из системного буфера в активное окно.
- `Alt+X` — отправить `SIGINT` в активный процесс.
- `Alt+-` — разделить активную панель на левую и правую.
- `Alt+=` — разделить активную панель на верхнюю и нижнюю.
- `Alt+Q` — закрыть активную панель, если она не последняя.
- `Alt+T` — создать новую вкладку.
- `Alt+,` — перейти на предыдущую вкладку.
- `Alt+.` — перейти на следующую вкладку.
- `Alt+W` — закрыть текущую вкладку, если она не последняя.
- `Alt+Shift+Q` — сохранить состояние и выйти из `mtrm`.
- `Left` / `Right` / `Up` / `Down` — передать стрелки в активный shell для перемещения курсора и навигации внутри программы.
- `Alt+Left` — перевести фокус влево.
- `Alt+Right` — перевести фокус вправо.
- `Alt+Up` — перевести фокус вверх.
- `Alt+Down` — перевести фокус вниз.
- `Shift+Up` / `Shift+Down` — прокрутить историю активной панели на одну строку.
- `Shift+PageUp` / `Shift+PageDown` — прокрутить историю активной панели на один экран.
- `End` — вернуть активную панель к живому хвосту.

По умолчанию встроенный keymap покрывает латинские символы, а значит уже подходит для английской, испанской и португальской раскладок, а также отдельно включает русскую раскладку, французский AZERTY и греческую раскладку для буквенных горячих клавиш.

Точный набор символов для буквенных горячих клавиш хранится в `~/.mtrm/keymap.toml` и может быть расширен под другие раскладки.

Если внешнее окно терминала теряет фокус, активная вкладка и рамка активной панели подсвечиваются красным.

## Каталог `~/.mtrm`

`mtrm` не требует отдельной настройки для хранения состояния.

При первом сохранении или первой загрузке keymap автоматически создается каталог:

```text
~/.mtrm
```

Внутри него хранятся как минимум:

```text
~/.mtrm/state.toml
~/.mtrm/keymap.toml
```

Положение прокрутки истории в панелях не сохраняется.

## Сценарий восстановления после перезапуска

1. Пользователь запускает `mtrm`.
2. Приложение либо создает новую среду, либо загружает `~/.mtrm/state.toml`.
3. Для каждой сохраненной вкладки восстанавливается раскладка.
4. Для каждого сохраненного окна запускается новая оболочка в том каталоге, который был записан в снимке.
5. Возвращаются активная вкладка и активное окно.

## Строка версии

`mtrm --version` печатает:

- последний локальный git tag, найденный во время сборки;
- через пробел время модификации текущего исполняемого файла в Unix seconds.

Это сознательно привязано к самому бинарю, а не к моменту выполнения `build.rs`, чтобы после реальной переустановки установленного файла версия менялась вместе с ним.
