# Порядок реализации

Реализацию вести в следующем порядке:

1. `crates/core`
2. `crates/config`
3. `crates/layout`
4. `crates/clipboard`
5. `crates/process`
6. `crates/input`
7. `crates/session`
8. `crates/state`
9. `crates/tabs`
10. `crates/ui`
11. `app`

Правила:

- начинать с независимых библиотек;
- не вводить циклических зависимостей между библиотеками;
- документацию по библиотеке писать только после того, как ее тесты проходят;
- если по ходу работы возникает необходимость в новой общей сущности, сначала проверять, не должна ли она жить в `core`.
