# Персональный терминальный менеджер (Rust)

1. Цель проекта
Создать консольное приложение для локальной (не удаленной!) работы, которое заменяет стандартные мультиплексоры (например tmux) и адаптирует управление под привычки пользователя (Windows/Web style).

2. Ключевой функционал (UX)

* Переопределение горячих клавиш:
* Ctrl+C: Копирование выделенного текста в системный буфер обмена.
   * Ctrl+V: Вставка текста из буфера.
   * Alt+X: Отправка сигнала прерывания (SIGINT / 0x03) в активный процесс.
* Управление пространством:
* Разбиение экрана на произвольное количество вертикальных и горизонтальных колонок.
   * Навигация между колонками с помощью клавиатуры (например, Alt + Стрелки).
* Автоматическое сохранение (Persistence):
* Приложение должно сохранять текущую структуру сплитов (сетку окон).
   * Для каждой колонки сохраняется путь (CWD — Current Working Directory), в которой она была открыта.

3. Техническая логика сохранения состояния

* Файл состояния: Вся конфигурация текущей сессии (координаты окон, их иерархия и пути) записывается в локальный файл (JSON/TOML) при каждом значимом событии (создание сплита, переход в другую папку).
* Восстановление: При запуске приложение читает файл и автоматически воссоздает структуру колонок, запуская в каждой новый экземпляр Shell и выполняя в нем переход в сохраненную директорию.
* История: Опциональное сохранение истории ввода для каждой конкретной колонки, чтобы контекст работы не терялся после перезапуска.

4. Архитектура и стек (Rust)

* Режим работы: Raw Mode через библиотеку crossterm. Это позволяет приложению перехватывать Ctrl+C до того, как его обработает ОС.
* Менеджер процессов: portable-pty. Создает псевдотерминалы для каждой колонки.
* Отрисовка интерфейса: ratatui. Используется для рисования рамок, разделения экрана и управления фокусом.
* Интеграция с ОС: arboard для работы с системным буфером обмена.

5. Особенности реализации

* Обработка ввода: Реализовать логический фильтр: если нажатие совпадает с «горячей клавишей» управления (копирование/сплит), оно обрабатывается логикой Rust. Если нет — транслируется в байты и отправляется в PTY активной колонки.
* Определение пути: Для сохранения состояния приложение должно уметь запрашивать у ОС текущую рабочую директорию дочернего процесса (через PID).

6. Эта программа НЕ является:

   - Клиент-серверным мультиплексором: В отличие от tmux или screen, здесь нет фонового сервера (daemon). Если вы закрыли окно терминала — все процессы внутри завершаются. Цель — сохранить структуру работы, а не живучесть процессов.
   - Инструментом для удаленной работы (SSH): Программа не проектируется для работы в условиях нестабильного интернета. Ей не нужно уметь «переподключаться» к сессии (detach/attach), так как работа ведется локально.
   - Эмулятором терминала в полном смысле: Она не рисует пиксели, не настраивает сглаживание шрифтов и не взаимодействует с видеокартой (GPU). Это TUI-оболочка, которая полагается на то, что внешний терминал уже запущен и умеет отображать текст.
   - POSIX-консерватором: Проект намеренно нарушает священные традиции Unix (вроде Ctrl+C для прерывания), отдавая приоритет привычкам пользователя современных ОС и браузеров.
